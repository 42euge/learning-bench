Skill Selection

Procedural Learning

What It Tests

Can the model learn to select the correct tool from a growing registry of tools with potentially misleading names and descriptions? This tests procedural learning in a realistic scenario—navigating a complex tool ecosystem where surface-level cues can be deceptive and only deep capability analysis reveals the right choice.

Three-Phase Framework

📝
Pre-Test
See a tool registry and task description, select the correct tool cold.
→
📖
Study
Analyze practice examples with correct selections and build a quick-reference guide.
→
✅
Post-Test
Select tool with own study notes + original registry.

Difficulty Dimensions

DimensionLevelsWhat Varies
Registry size 5, 15, 30 Number of tools. Larger registries = more distractors and context to parse.
Similarity distinct, confusable, adversarial Distinct: clear descriptions. Confusable: generic overlapping descriptions. Adversarial: deliberately misleading names/descriptions (capabilities are ground truth).
Seeds 0, 1, 2 Independent random generations.

Study examples scale with registry: 4 for 5 tools, 6 for 15, 8 for 30.
Total: 3 sizes × 3 similarity × 3 seeds = 27 items

Dataset Generation

Tools are generated on a fictional “Nexara Platform” with synthetic names (e.g., flux_bridge, data_weaver), descriptions, and capability lists across 15 domains. In adversarial mode, tool names and descriptions are swapped or made misleading while capabilities remain truthful. Practice examples include explicit explanations of why the correct tool was chosen.

Scoring

  • Metric: Post-learning accuracy (exact tool name match).
  • Extraction: Fuzzy match against valid tool names—exact match first, then substring on short lines.
  • Assertion: kbench.assertions.assert_equal(post_extracted, expected)

What Discriminates Models

  • 5/distinct: Easy baseline—5 clearly described tools, most models succeed.
  • 30/adversarial: The hardest setting. 30 tools with misleading names and descriptions. Only models that carefully read capabilities (not names) and learned from practice examples select correctly.
  • Study quality: Models that build accurate quick-reference guides mapping each tool to its actual capabilities outperform those that just summarize descriptions.
