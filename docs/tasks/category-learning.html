Category Learning

Concept Formation

What It Tests

Can the model discover a hidden classification rule from labeled examples and apply it to a novel object? This isolates concept formation—the ability to identify which features are relevant for categorization and form an explicit rule.

Three-Phase Framework

📝
Pre-Test
See labeled objects (size, color, pattern, shape) and classify a new one cold.
→
📖
Study
Analyze examples to identify the classification rule and verify it.
→
✅
Post-Test
Classify with own rule analysis + original examples.

Difficulty Dimensions

DimensionLevelsWhat Varies
Complexity single, two_feat, conditional Single: one feature determines category. Two-feature: conjunction of two features. Conditional: if-then chains with overrides.
Evidence few (4), mid (6), many (10) Number of labeled examples. Fewer = harder to disambiguate which features matter.
Seeds 0, 1, 2 Independent random generations.

Total: 3 complexity × 3 evidence × 3 seeds = 27 items

Dataset Generation

Objects are described by four features (size, color, pattern, shape) drawn from fixed pools. Classification rules are generated at three complexity levels. Training examples are sampled to be consistent with the rule, and the test object is guaranteed to be unambiguous under the true rule. Feature values are varied to avoid spurious correlations.

Scoring

  • Metric: Post-learning accuracy (exact category letter match, A–D).
  • Extraction: Last category letter found in the response via reverse line scan.
  • Assertion: kbench.assertions.assert_equal(post_extracted, expected)

What Discriminates Models

  • Single/many: Easy baseline—one feature, lots of examples.
  • Conditional/few: Requires identifying multi-step rules from minimal data with “ELSE IF” logic.
  • Feature distraction: All four features are always present, but only 1–2 are relevant. Models that latch onto irrelevant features fail.
