Rule Induction

Concept Formation

What It Tests

Can the model discover hidden transformation rules from input→output examples? Each item presents a set of word pairs produced by a secret rule composed of one or more string operations (Caesar shift, reverse, swap pairs, vowel shift, rotate half, consonant shift). The model must induce the rule and apply it to a new word.

Three-Phase Framework

📝
Pre-Test
See input→output examples and apply the same rule to a new word cold.
→
📖
Study
Hypothesize the exact rule (may be multi-step), verify against every example, revise if wrong.
→
✅
Post-Test
Apply the discovered rule using own study notes + original examples.

Difficulty Dimensions

DimensionLevelsWhat Varies
Complexity 1 op, 2 ops, 3 ops Number of chained string operations. 1-op rules (e.g., Caesar shift by 3) are straightforward. 2-op rules (e.g., reverse then vowel shift) require decomposing two transformations. 3-op rules require tracking three sequential operations.
Evidence 2, 4, 8 examples Number of input→output pairs shown. More examples make the rule easier to identify but also increase the cognitive load of verification.
Seeds 0, 1, 2 Independent random generations for statistical robustness.

Total: 3 complexity × 3 evidence × 3 seeds = 27 items

Dataset Generation

Rules are composed from a pool of six string operations: Caesar shift (by k), reverse, swap adjacent character pairs, vowel shift (a→e→i→o→u→a), rotate half (split at midpoint and swap halves), and consonant shift (by k positions in the consonant alphabet). For each item, a random subset of 1–3 operations is selected with random parameters, applied in sequence to common English words (e.g., “planet”, “bridge”, “castle”). The test word is held out from the examples, ensuring the model must generalize the rule rather than memorize specific mappings.

Examples

1-Op Rule (single transformation)

A single operation applied to each word:

“planet” → “tenalp”
“bridge” → “egdirb”
“castle” → “eltsac”
“forest” → “tserof”
Apply to: “garden”
Expected: nedrag

The rule is reverse. With 4 examples the pattern is fairly clear, but with only 2 examples the model has less evidence to confirm its hypothesis.

2-Op Rule (chained transformations)

Two operations applied in sequence:

“hunter” → “uhtnir”
“silver” → “islvir”
Apply to: “lemon”
Expected: (depends on specific seed)

The model must decompose two operations (e.g., swap adjacent pairs then vowel shift) and apply them in the correct order. Getting the order wrong produces a different output.

3-Op Rule (triple-chained transformations)

Three operations chained together. With only 2 examples, this is extremely hard:

“tunnel” → “???”
“winter” → “???”
Apply to: “purple”
Expected: (depends on specific seed)

Three chained operations (e.g., Caesar shift + reverse + consonant shift) create a large hypothesis space. With 8 examples, the model can verify candidate rules systematically; with 2, it often guesses wrong.

Scoring

  • Metric: Post-learning accuracy (exact transformed-word match).
  • Extraction: Last short line (≤30 chars), lowercased and stripped of quotes/punctuation.
  • Assertion: kbench.assertions.assert_equal(post_extracted, expected)

What Discriminates Models

  • 1-op / many examples: Most models handle a single transformation with 8 examples—this is the baseline. Failure here indicates fundamental issues with pattern matching.
  • Multi-op / few examples: 3 operations with only 2 examples is the hardest condition. The model must consider combinatorial possibilities and verify each hypothesis carefully.
  • Verification discipline: Models that systematically verify their hypothesized rule against every example before answering outperform those that guess after noticing a single pattern.
  • Operation order matters: Reverse-then-Caesar produces different output from Caesar-then-reverse. Models must track sequential composition, not just identify which operations are present.

Prior Work & References

Rule induction is one of the oldest paradigms in cognitive psychology, tracing back to Bruner’s concept attainment studies. Our task adapts this paradigm for LLMs using string transformations as the rule space, creating a domain where prior training knowledge provides no shortcut.

Cognitive Science Foundations

  • Bruner, Goodnow & Austin (1956). A Study of Thinking. Pioneered the concept attainment paradigm: learners classify instances to discover a hidden rule. Our task is a direct descendant—the model must induce the transformation rule from input–output examples.
  • Shepard, Hovland & Jenkins (1961). “Learning and memorization of classifications.” Established the difficulty ordering of six elemental classification types, showing that rule complexity (single-dimension vs. XOR vs. memorization) predicts human learning difficulty. Our complexity dimension (1/2/3 ops) similarly tests whether more complex rules are harder to induce.

Rule Induction & Hypothesis Testing in LLMs

  • Ainooson & Kunda (2025). “Analysis of Error Sources in LLM-based Hypothesis Search for Few-Shot Rule Induction”. Used LLMs in a hypothesis-driven search pipeline for the Abstraction and Reasoning Corpus (ARC). Found that hypothesis search achieves human-level performance while direct program generation falls behind—supporting our study-phase approach of explicit hypothesis formation.
  • Gao et al. (2024). “IDEA: Enhancing the Rule Learning Ability of Language Agent through Induction, Deduction, and Abduction”. Introduced RULEARN, a benchmark for rule-learning in interactive settings. Our task differs by using a pre/post framework to directly measure whether studying examples improves performance, rather than testing interactive exploration.
  • Srivastava et al. (2024). “WILT: A Multi-turn, Memorization-Robust Inductive Logic Benchmark for LLMs”. Inspired by the Wason 2-4-6 task, WILT tests iterative hypothesis refinement. Our task shares the inductive logic spirit but uses string transformations to create a clean, verifiable rule space.
  • Zhang et al. (2025). “From Reasoning to Learning: A Survey on Hypothesis Discovery and Rule Learning with Large Language Models”. A comprehensive survey unifying abduction, deduction, and induction as modes of hypothesis discovery. Our benchmark evaluates the inductive mode: forming general rules from specific examples.

In-Context Learning Foundations

  • Min et al. (2022). “Rethinking the Role of Demonstrations” (EMNLP 2022). Found that ICL often relies on format and label space rather than learning the actual input–label mapping. Our task specifically tests whether models can go beyond format-matching to induce genuine transformation rules.
  • Wei et al. (2023). “Symbol Tuning Improves In-Context Learning” (EMNLP 2023). Showed that standard LLMs struggle with arbitrary mappings. Rule induction from string transformations is a particularly demanding form of this capacity.
