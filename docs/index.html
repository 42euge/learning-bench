GenoBench-Learning

Do frontier models learn, or just recall?

A benchmark measuring whether LLMs can acquire and apply new knowledge—not just recall what they were trained on. Six cognitive sub-abilities, twelve tasks, all built on synthetic stimuli that cannot be memorized from training data.

View on Kaggle

Problem Statement

Why existing benchmarks miss this

Jagged intelligence is a frequent complaint about coding models but is difficult to articulate precisely—models excel at some tasks and fail at structurally similar ones, especially when the problem is genuinely new. Can we measure that gap in a practical way?

Agentic harness systems are multi-step pipelines where the critical failure mode isn't typically lack of knowledge, but the inability to effectively use and manipulate context to discover proper solutions. A model must do two things: externalize its understanding into a context (notes, observations, intermediate reasoning), and then effectively use that context to solve a downstream problem. These are separable skills, and current benchmarks measure neither.

GenoBench-Learning isolates both parts of this context-learning problem. The benchmark decomposes learning into two scored sub-problems: how well can a model construct a useful learned context from novel stimuli, and how well can it apply that context to solve a new instance? Each is measured independently and baselined against a one-shot approach.

Method

Pre/post study paradigm across all tasks

Every task follows the same three-phase framework. The model is tested before and after a structured self-study phase in which it must articulate the underlying rule in its own words. Learning gain—the delta between pre- and post-study accuracy—is the primary signal.

📝
Pre-Test
Answer cold. No study, no hints. Measures baseline.
→
📖
Study
Analyze material. Write notes, rules, mnemonics.
→
✅
Post-Test
Answer again with own study notes. Measures learning.

All stimuli are procedurally generated with random seeds, using invented vocabularies, grammars, and procedures that cannot exist in any training corpus. This guarantees that correct answers require genuine in-context learning, not memorization.

Tasks

Six cognitive sub-abilities, 12 tasks — 7 procedurally generated + 5 mined from real agent failures

🔗

Associative Learning

Paired Associate

Memorize and recall arbitrary nonsense-word-to-meaning pairs from a novel vocabulary. Tests rote associative binding—forming new arbitrary mappings during a conversation.

Associative Learning

Error Remediation

Learn the mapping from novel CLI error codes to correct remediation commands. An invented tool (zkl) with structured error families—models must learn the fix from examples, not guess.

27 items session mining

Concept Formation

Rule Induction

Discover a hidden classification rule from labeled examples and apply it to novel inputs. Scales from single-feature rules to conditional if-then chains with distractors.

27 items procedural generation
🧩

Concept Formation

Error Severity Triage

Classify output lines by severity: INFO, WARNING, RETRYABLE_ERROR, or TERMINAL_ERROR. Uses invented tool conventions—models must learn the tag system from labeled examples.

27 items session mining
🌍

Language Learning

Novel Grammar Induction

Induce the grammar of a completely novel language from example sentences, then judge whether new sentences are grammatical. Scales from fixed word order to nested agreement.

27 items procedural generation
👁

Observational Learning

Trace-Based Imitation

Learn a novel procedure by observing worked examples (execution traces on number lists), then apply it to new input. Scales from 1-step to 3+ step conditional branching.

27 items procedural generation

Observational Learning

Batch Error Diagnosis

Given build output with multiple errors, count the distinct root causes. Tests whether models diagnose all issues at once instead of fixing one at a time.

Procedural Learning

Skill Selection

Select the correct tool from a growing registry with misleading names. In adversarial mode, only deep analysis of capabilities reveals the right choice.

Procedural Learning

Novel Algorithm Execution

Execute a novel algorithm described in pseudocode on a given input. Tests whether the model can follow multi-step procedures it has never seen.

Procedural Learning

Stale State Recovery

Given a sequence of operations where state becomes invalid after mutations, identify where staleness occurs and insert the correct recovery step.

Infer reward distributions from a history of pulls and select the optimal arm. Tests whether models can learn value estimates from outcome data.

Results

Scaling ladder from 1B parameters to frontier

The benchmark reveals what standard evaluations cannot: not just whether a model knows things, but whether it can learn new things on demand, and whether explicit self-generated reasoning aids that learning.

Key Findings

  • Adversarial similarity is the pure discriminator. Distinct and confusable conditions hit ceiling (100%) for all models above 1B. Adversarial produces a clean gradient: 0% → 56% → 67% → 100%.
  • Small models can't follow the format. gemma-3-1b scores 33% on distinct (the easiest condition) and 0% at 15+ tools. It cannot reliably parse a tool registry regardless of difficulty.
  • Tool count compounds the difficulty. gemma-3-1b drops from 44% (5 tools) to 0% (30 tools). Even gemma-3-4b drops from 100% to 56–89%.
  • Learning gain requires sufficient baseline capability. Only gemini-2.5-flash shows meaningful positive gain (+7.4%). Models below a capability threshold produce study notes that are noise, not signal.
  • Negative learning gain is real. gemma-3-4b's post-learning accuracy is lower than pre-learning (−3.7%). Self-generated study notes introduced confusion—mirroring a known human learning phenomenon.
