2. **Change batching** — Given multiple related requirements, can the model group changes and validate once, rather than validate after each atomic edit? Tests planning and efficiency.

3. **Incremental refactoring signals** — After N incremental patches to the same file, can the model recognize when restructuring is needed? Tests metacognitive monitoring of code quality.

5. **Own-output tracking** — Can the model reason about files it recently created/modified without re-reading them? Tests whether the model treats its own outputs as known state vs. external state requiring verification.
